## **Eе можно реализовать по разному,но я покажу два вида**

1. ```cpp
    #include <vector>
    template<typname T>
       struct AdjacencyList{
            std::vector<std::vector<T>> adj;//то есть в ней списки индексов соседей i-ой ноды
       };
   ```
2. ```cpp
    #include <vector>
    template<typname T>
        struct Node {
            T value;
            std::vector<Node*> refs;//хранит указатели не тех соседей на которые есть ссылка
            //...
            /*
            я не стану писать проверку и другие методы,это даже не класс,а указатель те кто хотят,могут сами,ее будет много,все равно потом когда будем разбирать деревья,а пока просто introduction into the смежные списки и нелиненые структуры данных
            */
        };

   ```

### первое быстрее потому что алгоритмично без выкрутасов с инициалзациями,а просто матрица в котором храним значения индексов смежных вершин

### например A--ij[value]-->B--i+1j[value]-->A--ij+1[value]-->C
### index(C)--ret->(i+1,j--->foreach->n-1 -> index of other nodes)

## Примеры, где список смежности оптимален
### Разреженные графы
####
1. ***Когда рёбер мало по сравнению с вершинами (E << V²)***
2. ***Пример: социальные сети с миллионом пользователей, но каждый связан с десятками людей***
### Алгоритмы обхода графа
1. ***DFS / BFS***
2. ***Дейкстра, Беллман-Форд (поиск кратчайшего пути)***
3. ***Удобно и быстро проходить по списку соседей***
### Динамическое изменение графа
1. ***Добавление/удаление рёбер проще, чем в матрице***
2. ***Например, в игровых движках для сетевой логики или карт***
### Сложные объекты в вершинах (OO-подход)
1. ***Когда вершины имеют состояние и методы***
2. ***Node\* с RAII позволяет аккуратно управлять связями и памятью***
