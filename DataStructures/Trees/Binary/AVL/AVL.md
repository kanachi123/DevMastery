# ***AVL***

## **Это BST(binary search tree) которое сбалансировано по высоте так,что для каждой его вершины высота её двух поддеревьев различается не более чем на 1**

```
                 5
                / \
|--------------3   7 -| h_right
|-----h_left--/        
|------------2

```
$h_{left}$ = 2{3,2}  
$h_{right}$ = 1{2}  
sub == 2 - 1 = 1

$|h_{left} - h_{right}| \leq 1$  

## ***Balance-Factor(BF)***
## *разница высот левого и правого поддерева может быть*
1. 0
2. 1
3. -1

```
    5
     \
      7
       \
        9
```        
*здесь разница 0 - 2 = |-2| > 1,*
*а если высота больше поиск будет долгим это не оптимально*
*в отличие от BST,AVL делает поворот так чтобы разница была меньше*
*либо равно 1*

*например родитель становится ребенком*
```  
      7
     / \
    5   9
```
*вот еще*
```
      T
     / \
    E   W
   / \
  O   D
 /
P
```
*Левая часть выше правой → нужно правое вращение вокруг T*

*поднимается левый ребёнок*
```
      E
     / \
    O   T
   /
  P
```
*перераспределяем остальные вершины*

*Поддерево D должно остаться справа от E, но слева от T*

*E < D < T*
```
      E
     / \
    O   T
   /   / \
  P   D   W

```

## *Идея поворота*

## *При вращении:*

1. *один узел поднимается вверх*

2. *другой опускается вниз*

3. *среднее поддерево переезжает*

***Но ни один элемент не удаляется***

## ***4 типа поворотов (это самое важное)***

### ***Right rotation (LL)***

### *Когда левое поддерево слишком высокое*
```
    A
   /
  B
 /
C
```

   |
   v


### ***Left rotation (RR)***

### *Когда правое поддерево слишком высокое*
```

 \

   \
    C

```
### *После:*
```
   B
  / \
 A   C
```
### ***Left-Right rotation (LR)***
```
    A
   /
  B
   \
    C
```
Делается 2 поворота:
1. left
2. right

### ***Right-Left rotation (RL)***
```
A
 \
  B
 /
C

```
## *Делается два поворота:*
1. right
2. left

| операция | сложность |
| -------- | --------- |
| поиск    | O(log n)  |
| вставка  | O(log n)  |
| удаление | O(log n)  |

## *обычно в каждом узле хранят*

1. key
2. left
3. right
4. height

## *Когда происходит балансировка*

### *Балансировка выполняется после:*

1. вставки

2. удаления

Алгоритм:
1. вставили узел
2. идём вверх к корню
3. считаем баланс
4. если |BF| > 1 → делаем поворот

| BST                 | AVL                   |
| ------------------- | --------------------- |
| может стать списком | всегда сбалансировано |
| поиск O(n)          | поиск O(log n)        |
| нет поворотов       | есть повороты         |
